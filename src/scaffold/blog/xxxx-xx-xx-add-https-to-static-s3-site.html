    Add HTTPS to Website Hosted in S3

    xxxx/xx/xx
    Tags: Website Hosting HTTPS AWS S3

    Introduction

    This post goes over the steps to add HTTPS to a website hosted in S3. It's assumed that you have already hosted a static website in S3 and have acquired the domain via Route 53. For information on how to do this see Hosting a Static Website in AWS S3.

    There are three steps to this process: requesting a certificate, creating a CloudFront distribution, and updating the A records in Route 53.

    These steps need to be done for both the www and non-www (root) domains.

    Step 1: Requesting a Certificate

    • This needs to be in US East (N. Virginia) Region for CloudFront. If this region isn't selected then you won't be able to select the certificate when creating the CloudFront distribution nest.
    • Go to the Certificate Manager in AWS.
    • Click on "Request a certificate".
    • Enter the domain name for the certificate.
    • Choose the DNS validation method.
    • Click xxx
    • The certificate will be in a pending status, when its done it will be in a xxxx status

    Step 2: Creating a CloudFront Distribution

    • Go to the CloudFront service in AWS.
    • Click on "Create Distribution".
    • Under "Origin" for "Origin Domain" choose the S3 bucket where the static website is.
    • For the setting unders Default Cache Behaviour Seetings, under Viewer, set Viewer protocol policy to Redirect HTTP to HTTPS
    • In the "Web Application Framework (WAF)" section choose to either enable or disable AWS WAF security protections.
    • For the fields under "Settings":
      • Choose "Add Item" for "Alternate domain cname (CNAME) - optional" and enter your subdomain.
      • For "Custom SSL Certificate" choose the certificate created.
    • Click "Create Distribution"
    • After CloudFront creates your distribution, the value of the Status column goes from In Progress to Deployed. This can take a few minutes.
